MotionFlix

HTML | JavaScript | CSS

Project Description

  • Final project for CS:2500 in Spring 2024
  • Basis: a dataset of the top 1000 movies on IMDb
  • Local website developed with two other classmates
  • General concept: movie education
    • Can test and expand current movie knowledge
  • What we did:
    • Usability testing, interviewed stakeholders
    • Created low-fidelity prototypes
    • Programmed our website from scratch

My Contributions

  • Kept group organized and maintained effective documentation through each project phase
  • Browsing Page
    • Wrote JavaScript functions to read the dataset and filter it (genre, IMDb rating, search bar)
    • Displayed filtered data with HTML and Bootstrap
  • Trivia Page
    • Implemented: General easy, medium, hard levels
      • Movie-themed questions I wrote myself

What I Learned:

How to create user stories and perform competitive assessments.

How to design low-fidelity wireframes.

How to implement a high-functioning prototype.

And finally - how to utilize user testing to inform future phases of a project.

PART 1:

USER PERSONAS AND REQUIREMENTS

  • Brainstorming an application related to film
  • Identifying potential target users
  • Conducting interviews with said users to establish requirements
  • Creating user personas
    • Based on the individuals we interviewed
  • Completing a competitive assessment with multiple other applications
    • I analyzed Letterboxd
  • Outcomes
    • A well-established idea with a defined group of target users

MUST HAVE

  • A simplistic structure and design
    • Which will allow users to more efficiently and effectively use the app
  • An interface that one can use without much of a learning curve
    • A strong sense of learnability
  • An aspect of interactivity to keep users engaged
    • Interactive filtering/quiz features
  • Efficiency when it comes to finding information or accessing different componenets of the app
  • A strong sense of functionality

COULD HAVE

  • A way for users to keep track of movies that they have seen or would like to see
    • Additional interactive component + more personalized experience
  • A progress bar for quizzes so users know how far they are
    • To prevent user frustration
  • Illustrations that can enhance the visual aspect of the pages
    • To keep users engaged

SHOULD HAVE

  • A casual tone when portraying different kinds of information
    • For a wide audience of viewers
  • Have a leisurely atmosphere so as to prevent any pressure on users
    • Would make a more supportive and collaborative environment overall
    • Would enhance users' educational experiences

PART 2:

LOW-FIDELITY PROTOTYPE

  • Individual + Group Design of Low-Fidelity Prototype
  • Cognitive Walkthrough: Four key tasks with each user persona
    • Searching for a movie to learn more about
    • Adding a movie to the watchlist
    • Searching for a movie sorted as an action film with the rating going from decreasing to increasing
    • Searching for trivia they can play
  • Outcomes
    • Valuable explorative results to use for future phases in the project

PART 3:

HORIZONTAL AND VERTICAL PROTOTYPE

  • Implementing the prototype with code
  • Making decisions about features
  • Providing a descriptive walkthrough
  • Outcomes
    • A fully functioning local website that we can test with potential users!

PART 4:

USABILITY TESTING

  • Conducting usability tests
  • Collecting relevant data about usability issues
    • Use of NASA-TLX and Satisfaction Questionnaire
  • Re-analyzing four main tasks from Phase Two
    • Completion times and effectiveness of tasks
  • Documenting possible solutions to usability issues
  • Reflecting on the process - what went will, what didn't, possibilities of going live
  • Outcomes
    • A strong understanding of how to conduct user testing
    • A newfound knowledge of how effective our application really was
